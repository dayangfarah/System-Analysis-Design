# SECP 2613: System Analysis and Design

## 📌 Course Overview
* **Faculty:** Faculty of Computing
* **Program Name:** Bachelor Computer Science (Data Engineering) with Honors
* **Course Code:** SECP 2613
* **Academic Session/Semester:** 2023-2024/1
* **Credit Hours:** 3

---

## 📖 Course Synopsis
The main focus of this course is to provide a practical approach of systems analysis and designing skills for the students using structured and object-oriented methodology. Hence the course enables students to study information system requirements for any system application within an organizational context. 

The contents are sequentially organized directly from planning, analysis, designing and implementation phases. From the resulting output of the planning and analysis phase shall enable students to form input, output and interface design. Hence a prototype design can be demonstrated.

## 💭 Reflection

### What I Learned
Taking SECP 2613 gave me a complete, end-to-end perspective on how software applications are built in the real world. I moved from simply writing code to understanding the entire Software Development Life Cycle (SDLC). Learning how to model systems using Data Flow Diagrams (DFDs) and Unified Modeling Language (UML) diagrams-like Use Case, Sequence, and Activity diagrams-taught me how to visually map out complex technical logic before a single line of code is written. 

### Challenges & Growth
The Work Based Learning (WBL) approach was both challenging and incredibly rewarding. Managing a real-world case study forced my team to balance user requirements, design architecture, and system testing under actual project constraints. Navigating through structured vs. object-oriented methodologies required a massive shift in mindset. Transitioning from abstract requirements to practical designs-like interface prototyping and process specifications (decision trees and tables)-significantly improved my problem-solving and systems-thinking abilities.

### Future Application
As an aspiring data engineer and software professional, the blueprinting skills I developed here are foundational. I will use these analysis and design methodologies to architect cleaner data pipelines, gather precise client requirements, and create comprehensive documentation. Understanding the structural impacts on information systems will allow me to build scalable, professional software configurations that seamlessly integrate into any organization’s technical environment.
